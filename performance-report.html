Interventions

01 Remove unused tailwind.js Done

The full Tailwind CSS runtime engine (398 KB / 123 KB gzipped) was in assets/js/ but never referenced from any layout, include, or page. Pure dead weight on every page load. Deleted.

-398 KB raw · -123 KB gzipped per page

02 Non-render-blocking Google Fonts Done

Converted the Google Fonts <link rel="stylesheet"> to <link rel="preload" as="style" onload="this.rel='stylesheet'"> with <noscript> fallback. This unblocks First Contentful Paint — the browser no longer waits for Google's CSS before rendering the page.

Unblocks FCP · ~200-500ms improvement on slow connections

03 Cloudinary auto-format + compression Done

Added f_auto,q_auto transforms to all Cloudinary image URLs (cookie banner mascot, footer stamps, 404 image, og:image). Cloudinary now auto-serves WebP/AVIF with quality optimization. Also added width constraints (w_240, w_400, w_600) based on display size to prevent oversized downloads.

~60-80% image size reduction · WebP/AVIF auto-served

04 Extract inline CSS/JS from includes Deferred

cookie-banner.html (315 lines) and glass-cursor.html (224 lines) inline ~12.5 KB of CSS+JS into every page. Extracting to external files would make them cacheable across navigations, saving ~3 KB gzipped per page after the first load. Deferred due to moderate complexity (touch/swipe handlers, animation loops) and low relative impact vs. other fixes.

Potential: -3 KB/page (cached) · Deferred

05 Remove .woff font fallbacks Done

Removed 4 .woff files (124 KB total) that served as fallbacks for IE11 and old Android. All modern browsers support .woff2. Also removed the non-critical Bold font preload (only Journal + Regular needed for first paint).

-124 KB static assets · -24 KB preload budget

Future Recommendations

Responsive Images with srcset

Article images currently have no srcset or sizes attributes. When real articles are published with Cloudinary images, use Cloudinary URL transforms to serve responsive variants:

BreakpointWidthURL Transform
Mobile400ww_400,f_auto,q_auto
Tablet800ww_800,f_auto,q_auto
Desktop1200ww_1200,f_auto,q_auto

Self-Host Google Fonts

Inter is loaded from fonts.googleapis.com, which requires a DNS lookup + connection to Google's CDN. Self-hosting the font files (already partially done with woff2 infrastructure) would eliminate this external dependency and allow the CSS to reference local files with no additional connection.

HTML Minification

HTML output is unminified (~1300 lines per article page). A Jekyll plugin like jekyll-compress-html could reduce HTML size by ~10-15%. Low priority given gzip already handles most of the savings.

Digital Ocean App Platform Configuration

When migrating from GitHub Pages:

  • Enable gzip/brotli compression in the CDN layer
  • Set Cache-Control: public, max-age=31536000, immutable for /assets/ (fonts, CSS, JS)
  • Set Cache-Control: public, max-age=3600 for HTML pages
  • Move CSP from <meta> tag to HTTP header
  • Add X-Content-Type-Options: nosniff and Strict-Transport-Security headers
